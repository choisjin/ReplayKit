모듈 가이드ReplayKit에서 사용 가능한 모든 모듈의 함수 및 파라미터 가이드

이 문서는 ReplayKit의 모듈 명령(module_command) 스텝에서 사용할 수 있는 모든 모듈과 함수를 설명합니다. 코딩 지식 없이도 각 함수가 어떤 장비를 제어하고, 어떤 값을 넣어야 하는지 확인할 수 있습니다.

앱 내 가이드 스텝 추가 시 함수를 선택하면 설명이 자동으로 표시됩니다. 이 문서는 전체 목록을 한눈에 보거나 인쇄하여 참고하기 위한 것입니다.

StopLogging DLT 캡처 종료 + 메모리 버퍼를 파일로 일괄 저장. 자동 경로 또는 지정 경로. save_path: 저장 파일 경로. 빈 값이면 자동 (재생 중: run_dir/logs/dlt_*.log, 스텝 테스트: backend/results/Temp_logs/dlt_*.log)
count_keyword 로그에서 keyword가 등장할 때마다 실시간 카운트하고 발생 시간을 누적. 같은 name으로 첫 호출=시작, 재호출=결과 반환. 통계 용도 (시나리오 fail에 영향 없음). keyword: 찾을 부분 문자열. 예: 'Pass', 'BootComplete'
name: 카운터 식별자. 빈 값이면 keyword 자체. 같은 name 재호출 시 결과 조회
reset_count_keyword count_keyword 카운터를 리셋합니다. name 빈 값이면 모든 카운터 제거. name: 리셋할 카운터 이름. 빈 값이면 전체
assert_keyword 캡처되는 모든 라인이 keyword를 **포함해야** 정상. **미포함 라인이 들어오면 fail step row 자동 누적**. 정상 키워드(예: 'OK', 'Pass', 'BootComplete')를 단언할 때 사용. 비정상 키워드 검출은 fail_on_keyword 사용. keyword: 라인에 반드시 포함되어야 하는 부분 문자열. 예: 'Pass', 'OK'
name: 단언 식별자. 빈 값이면 'assert_<keyword>'. 같은 name 재호출 시 miss 통계 반환
reset_assert_keyword assert_keyword 단언을 리셋합니다. name 빈 값이면 모든 단언 제거. name: 리셋할 단언 이름. 빈 값이면 전체
fail_on_keyword 캡처되는 라인에 keyword가 **포함되면 fail row 누적**. assert_keyword의 반대. 'ERROR'/'Fail'/'crash' 등 비정상 키워드 검출용. **time>0이면 동기 모드** (해당 시간 동안만 모니터링 후 자동 종료, 검출 fail은 이 스텝 바로 아래 Fail_Count_N으로 인라인 표시). time=0이면 백그라운드 모드. keyword: 검출할 부분 문자열. 이 단어가 라인에 들어오면 fail로 보고. 예: 'Fail', 'ERROR'
time: 동기 모니터링 시간(초). >0이면 해당 시간 동안만 감시 후 자동 unregister + 다음 스텝으로. 0(기본)이면 백그라운드.
name: 검출기 식별자. 빈 값이면 'fail_<keyword>'. 같은 name 재호출 시 hit 통계 반환
reset_fail_on_keyword fail_on_keyword 검출기를 리셋합니다. name 빈 값이면 모든 검출기 제거. name: 리셋할 검출기 이름. 빈 값이면 전체
StartSave DLT 데몬에 연결하고 로그 캡처 + 파일 저장을 시작합니다. (연결 → 캡처 → 저장 한번에 수행) save_path: 저장 파일 경로. 빈 값이면 backend/logs/에 자동 생성
StopSave 로그 파일 저장을 중단하고 DLT 연결을 해제합니다. (없음)
MarkStep 현재 로그 위치에 스텝 경계를 표시합니다. SearchRange로 구간 검색할 때 사용합니다. step_index: 스텝 인덱스 번호
SearchAll 처음부터 현재까지 전체 로그에서 키워드를 검색하여 PASS/FAIL 판정합니다. keyword: 검색 키워드 (공백 구분 시 AND 조건). 예: 'BootComplete', 'ERROR crash'
count: 최대 매칭 결과 수 (기본 5)
SearchRange MarkStep으로 표시된 스텝 구간 내에서 키워드를 검색하여 PASS/FAIL 판정합니다. keyword: 검색 키워드 (공백 구분 시 AND 조건)
from_step: 시작 스텝 인덱스
to_step: 종료 스텝 인덱스
count: 최대 매칭 결과 수 (기본 5)
WaitLog 키워드가 포함된 로그가 나타날 때까지 대기합니다 (블로킹). 타임아웃 시 FAIL 반환. keyword: 검색 키워드 (공백 구분 시 AND 조건)
timeout: 최대 대기 시간 (초, 기본 30)
ExpectFound 키워드가 나타날 때까지 전체 로그를 주기적으로 검색합니다. 즉시 검색 후 없으면 간격(timeout/max_retries)마다 재시도. 발견 시 PASS, 끝까지 없으면 FAIL. keyword: 검색 키워드 (공백 구분 시 AND 조건)
timeout: 총 대기 시간 (초, 기본 60)
max_retries: 최대 재시도 횟수 (기본 5). 간격 = timeout ÷ max_retries
ExpectNotFound 키워드가 끝까지 없는지 전체 로그를 주기적으로 확인합니다. 즉시 검색 후 발견 시 즉시 FAIL. 없으면 간격(timeout/max_retries)마다 재확인. 끝까지 없으면 PASS. keyword: 검색 키워드 (공백 구분 시 AND 조건)
timeout: 총 확인 시간 (초, 기본 60)
max_retries: 최대 확인 횟수 (기본 5). 간격 = timeout ÷ max_retries
GetStatus 현재 모듈 상태를 조회합니다 (연결, 캡처, 저장, 스텝마크 등). (없음)
ClearLogs 로그 버퍼와 스텝 마킹을 초기화합니다. (없음)

시스템 & 유틸리티

CMD

명령어 실행 — Windows 명령 프롬프트(CMD) 명령을 실행하는 모듈. 외부 프로그램 실행이나 시스템 명령에 사용
함수설명파라미터
Run 명령어를 실행하고 완료될 때까지 기다립니다(블로킹). 실행 결과를 반환합니다. command: 실행할 명령어. 예: 'ping 127.0.0.1 -n 3', 'ipconfig'
timeout: 최대 대기 시간 (초). 기본값: 300초
Check 명령어를 실행하고 출력 결과를 기대값과 비교합니다(블로킹). 실패 시 'FAIL:' 접두사를 반환하여 스텝이 fail로 판정됩니다. command: 실행할 명령어
expected: 출력에 포함되거나 완전히 일치해야 할 기대 문자열
match_mode: 비교 방식: 'contains'(부분 일치) 또는 'exact'(완전 일치). 기본값: contains
timeout: 최대 대기 시간 (초). 기본값: 300초
RunCapture 명령어를 백그라운드로 실행하고 즉시 '[BG_TASK:bg_x]' placeholder를 반환합니다(논블로킹). 폴링 엔드포인트를 통해 나중에 실제 결과를 회수할 수 있습니다. command: 실행할 명령어
CheckCapture 명령어를 백그라운드로 실행하고 '[BG_TASK:bg_x]' placeholder를 반환합니다. 완료 시 서버가 expected/match_mode로 최종 pass/fail을 판정합니다. command: 실행할 명령어
expected: 기대 문자열
match_mode: 'contains' 또는 'exact'. 기본값: contains
RunBackground 명령어를 서브프로세스로 실행합니다(fire-and-forget). PID가 반환되지만 결과는 회수할 수 없습니다. command: 실행할 명령어. 예: 'notepad', 'python script.py'
Kill 백그라운드 프로세스를 종료합니다. pid: 종료할 프로세스 PID
ListBackground 실행 중인 백그라운드 프로세스 목록을 반환합니다. (없음)
