SerialLogging

Serial Logging — Captures and saves logs via serial port connection, with keyword-based fail detection. **⚠️ Usage: StartLogging and StopLogging MUST be used as a pair.** - If you call StartLogging without StopLogging: logs are NOT saved to file, and the serial port stays open, which can cause port-busy errors on the next run. - Place StartLogging at the beginning of your scenario and StopLogging at the end. - Cleanup will auto-call StopLogging if the scenario aborts, but you should still call it explicitly in the normal flow. Standard usage pattern: 1) StartLogging() ← first step 2) SendCommand / SendCommand_*_on_keyword ... ← middle steps 3) StopLogging() ← last step (writes log file + releases port)
FunctionDescriptionParameters
StartLogging Connects to the serial port and starts capture (memory only, no file save). Viewer modal opens automatically. The port is stabilized and capture thread is ready by the time this returns. **⚠️ Must be paired with StopLogging.** Without StopLogging, the in-memory buffer is never flushed to disk and the serial port stays open. Always call StopLogging at the end of the scenario. settle_ms: Port stabilization wait after open (ms). Default 500. Ensures USB-Serial driver reset/buffer settle so the next step's SendCommand isn't dropped. Use 1500~2000 for DTR-reset boards (Arduino), 100~200 for fast industrial serial.
StopLogging Stops serial capture, writes the memory buffer to a file, and releases the serial port. Auto path or explicit path. **⚠️ Must be paired with StartLogging.** This call is what: • Flushes the in-memory log buffer to a .log file on disk • Closes the serial port so it can be reused next run • Tears down the capture thread to avoid resource leaks Even if the file-save step throws, the finally block guarantees the port is released. save_path: Save file path. Auto if empty (during playback: run_dir/logs/serial_*.log, step test: backend/results/Temp_logs/serial_*.log)
SendCommand Sends a text command via the serial port. command: Command to send. e.g. 'reboot', 'cat /proc/version'
encoding: Encoding (default: utf-8)
append_newline: Auto-append newline. true/false (default: true)
SendCommand_fail_on_keyword Sends a command, then **FAILS if the keyword appears** in the response. Used to detect abnormal keywords like 'ERROR'/'Fail'/'crash'. After sending, monitors only newly captured lines for 'time' seconds; every matching line is appended as a fail row and displayed inline (Fail_Count_N). Returns PASS if no match. command: Command to send. e.g. 'status', 'self_test'
keyword: Substring that triggers FAIL. e.g. 'ERROR', 'Fail', 'crash'
time: Response monitoring duration (seconds). Default 5
encoding: Encoding (default: utf-8)
append_newline: Auto-append newline. true/false (default: true)
SendCommand_pass_on_keyword Sends a command, then **PASSES if the keyword appears** in the response. Used for normal-response keywords like 'OK'/'Pass'/'BootComplete'. Returns PASS immediately on first match (early exit). If not found within 'time' seconds, appends one fail row and returns FAIL. command: Command to send. e.g. 'ping', 'getversion'
keyword: Substring required for PASS. e.g. 'OK', 'Pass', 'BootComplete'
time: Response wait duration (seconds). Default 5
encoding: Encoding (default: utf-8)
append_newline: Auto-append newline. true/false (default: true)

DLTLogging

DLT Logging — Captures and saves logs via direct TCP connection to the DLT daemon, with keyword-based PASS/FAIL evaluation (no GUI required)
FunctionDescriptionParameters
StartSave Connects to the DLT daemon and starts log capture + file saving in one step (connect -> capture -> save). save_path: Save file path. Auto-generates in backend/logs/ if empty
StopSave Stops log file saving and disconnects from the DLT daemon. (none)
MarkStep Marks a step boundary at the current log position. Used with SearchRange for range-based searching. step_index: Step index number
SearchAll Searches the entire log from the beginning to the current position for a keyword, returning PASS/FAIL. keyword: Search keyword (space-separated = AND condition). e.g. 'BootComplete', 'ERROR crash'
count: Maximum number of matching results (default: 5)
SearchRange Searches for a keyword within the step range marked by MarkStep, returning PASS/FAIL. keyword: Search keyword (space-separated = AND condition)
from_step: Start step index
to_step: End step index
count: Maximum number of matching results (default: 5)
WaitLog Waits (blocking) until a log containing the keyword appears. Returns FAIL on timeout. keyword: Search keyword (space-separated = AND condition)
timeout: Max wait time in seconds (default: 30)
ExpectFound Periodically searches the entire log until the keyword is found. Searches immediately, then retries at intervals (timeout / max_retries). Returns PASS if found, FAIL if not found after all retries. keyword: Search keyword (space-separated = AND condition)
timeout: Total wait time in seconds (default: 60)
max_retries: Maximum retry count (default: 5). Interval = timeout / max_retries
ExpectNotFound Periodically checks the entire log to confirm the keyword is absent. Returns FAIL immediately if found. Rechecks at intervals (timeout / max_retries). Returns PASS if not found after all checks. keyword: Search keyword (space-separated = AND condition)
timeout: Total check time in seconds (default: 60)
max_retries: Maximum check count (default: 5). Interval = timeout / max_retries
GetStatus Returns the current module status (connection, capture, save, step marks, etc.). (none)
ClearLogs Clears the log buffer and step markings. (none)
